# Security Policy

## Supported Versions

Security updates are provided for the current development line (`v0.x`).

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities by emailing: `security@audience-engine.local`

Include:
- A clear description of the issue and affected component(s).
- Reproduction steps or proof-of-concept.
- Impact assessment if known.
- Suggested remediation (optional).

## Response Expectations

- Initial triage response target: within 3 business days.
- Status update cadence: at least once every 7 days until closure.
- Coordinated disclosure is expected after a fix is available.

## Scope Notes

- PII handling, policy gating, and export path controls are in-scope.
- Third-party infrastructure vulnerabilities should also be reported here when they affect this project's threat model.
